## 0. The Trash bug (reported: icon always empty)

**Root cause (two stacked defects), `Jetty/Dock/DockModel.swift:251-311`:**

1. The empty/full probe enumerates `~/.Trash` and every mounted volume's
   `.Trashes/<uid>` with `opendir`/`readdir`. Since macOS 10.15, enumerating the
   Trash is TCC-gated — without Full Disk Access the `opendir` fails with EPERM,
   the probe returns `.unreadable`, and the aggregate state becomes `.unknown`.
2. Commit `41d4b62` ("Fix false-full Trash icon state") flipped the fallback:
   `.unknown` now maps to the **empty** icon. So on any machine where Jetty lacks
   Full Disk Access (i.e. virtually every install — Jetty never requests FDA), the
   tile is *stuck on empty forever*, no matter what the Trash contains.

A third latent defect: the icon comes from `NSImage(named: NSImage.trashFullName /
trashEmptyName)`. These legacy named images are not a stable API surface on modern
macOS; when they fail to resolve the tile falls back to the SF Symbol `trash`
(`DockTileView.fallbackSymbol`), which is — you guessed it — an *empty* can.

**Fix [#46]:** take the Trash icon from `NSWorkspace.shared.icon(forFile:)` on the
user's Trash URL. IconServices/Finder maintains that icon in the correct empty/full
state with **no permissions required** — it is the canonical third-party-dock
approach and sidesteps both the TCC probe failure and the named-image fragility.
This also let us delete the per-rebuild synchronous filesystem scan from
`DockModel.rebuild` (see P1 below): the model caches the workspace icon and
`TrashMonitor.onChange` → `invalidateTrashIcon()` (previously an empty no-op!)
clears the cache, so the can flips live when items are trashed/emptied.

Residual: the pure `trashState(at:)` probe is kept (unit-tested) for future use
(e.g. an "N items" accessibility value when the probe is readable), but no longer
gates the icon. True per-item counts still need FDA and are deliberately not used.

## 1. Bugs (correctness)

### Critical / High

- **B1 [#46] Trash icon always empty** — see §0.
- **B2 [#51] Destructive confirmation alert can render behind the Jetty Menu.**
  `JettyMenuController.swift:117-145`: the menu panel sits at `.popUpMenu` and stays
  ordered-in while `alert.runModal()` runs; the "Shut Down / Restart / Empty Trash?"
  alert (modal level) can be fully hidden behind it, with Return bound to the
  destructive default button. Fix: order the menu out before `runModal`, restore
  after; also raise the updater alert above the menu (`UpdateChecker.swift:228-236`,
  which explicitly sets `.floating` — *below* the menu — and whose modal interlock
  freezes the menu).
- **B3 [#51] Synchronous AppleEvents on the main thread.** `MenuCommand.swift:50-54`,
  `PowerCommands.swift:99-105`: first-run Automation consent blocks Jetty's whole UI;
  `tell application "Finder" to empty the trash` blocks as long as Finder takes.
  Fix: run scripts on a background queue (fire-and-forget, errors logged), matching
  the pattern already used elsewhere.
- **B4 [#47] Hover labels are clipped — invisible at defaults, 100% broken on top
  docks.** `DockTileView.swift:227-239` floats the label `0.75 × base` above the
  tile, but the panel's across headroom is only `(mag−1) × iconSize` (0 when
  magnification is off) and the container clips to bounds (`DockPanelController:103`).
  With magnification ≤ ~1.56× the capsule is shaved or entirely outside the window;
  on **top-edge** docks the hard-coded "above" direction is always off-panel; on
  vertical docks it sits over the previous tile. Fix: edge-aware label placement
  (bottom→above, top→below, left→trailing, right→leading) + label headroom in
  `contentSize()` when `showLabels` is on.
- **B5 [#48] Dock auto-hides underneath its own popovers.** `setInteractionHeld`
  is wired only for context menus (`DockController.swift:333-335`). With Window Peek
  or a folder stack open, crossing the gap from tile to popover fires the hide timer
  — the dock slides away mid-interaction and snaps back: a flicker loop. Fix: hold
  the dock while either popover is open (new `onClose` callbacks on
  `WindowPeekController`/`FolderStackController`).
- **B6 [#53] Window Peek does main-thread AX IPC with no messaging timeout.**
  `AppWindows.swift:109-126` never calls `AXUIElementSetMessagingTimeout`; hovering
  a beach-balling app's tile can freeze Jetty ~6 s per window (the responsiveness
  monitor already uses a 0.5 s timeout — `AppResponsivenessMonitor.axProbe`).
  Also, `WindowPeekController.show()` enumerates the window list synchronously and
  `WindowPeek.load()` immediately re-enumerates the same list — the hover pays
  twice. Fix: 0.5 s timeout, pass the hover-time list into the model, only fetch AX
  titles when a CGWindowList title is missing.
- **B7 [defer] Update asset never authenticated.** `UpdateDownloader.swift:26-39`
  trusts HTTPS + unauthenticated GitHub JSON; no size/signature/Team-ID check before
  revealing the DMG in Finder. (REVIEW C1 — needs a signing story; out of scope here.)
- **B8 [defer] First launch can hide the system Dock without consent.** (REVIEW H1 —
  needs onboarding UI design; out of scope.)

### Medium

- **B9 [#49] Magnification snap-off in the headroom.** `DockView.swift:264-269`:
  `.onContinuousHover` is attached to the resting-height slot stack only; moving the
  pointer up a magnified icon past resting height fires `.ended` → everything snaps
  to 1× → re-entering re-fires `.active`. Visible stutter. Fix: track hover on the
  full-panel container.
- **B10 [#49] Active-app glow dots misaligned in overflow.** `DockView.swift:180-193`
  adds `magnificationAlongExtra/2` lead even when magnification is suspended in the
  overflow-scroll state, so every glow sits ~20 pt ahead of its tile.
- **B11 [#50] Menu keyboard nav recenters on every arrow press.**

  slides on each keypress. Spotlight scrolls minimally. Fix: no anchor.
- **B12 [#54] Folder-stack Escape monitor eats Escape from other Jetty windows and
  can never dismiss the stack.** `FolderStackController.swift:134-137`: the
  borderless non-activating panel can never be key, so the local monitor only fires
  when the Jetty Menu/Settings is key — swallowing their Escape. Fix: remove it
  (outside-click + hover-leave already dismiss).
- **B13 [#54] Re-opening Settings while miniaturized does nothing visible.**
  `SettingsWindowController.swift:54-57`: `makeKeyAndOrderFront` doesn't
  deminiaturize. Fix: `deminiaturize` first.
- **B14 [#54] "Restore System Dock" is a dead click when Jetty isn't managing the
  Dock.** `AppDelegate.swift:165-167`, `GeneralView.swift:20-27`: the restore

- **B15 [#54] Toggle-hide hotkey instantly undone by edge hover.** Pressing the

  behavior).
- **B16 [#54] A panel created under a resting pointer stays hidden.** Hot-plug a
  display while the cursor sits at its edge: nothing replays the pointer position,

- **B17 [defer] Failed hotkey registration is silent.** `CarbonHotkey.register`

- **B18 [#52] Weather: no request timeout, invalid coordinates, missing

- **B20 [defer] Bookmark resolution can block the main thread and "succeeds" for

- **B21 [defer] Edge drop-sensor steals file drags aimed at other apps.**

- **B22 [defer] `TileAccent` cache keyed by tile id only** — stale glow after a
  custom-icon change; unbounded. (`TileAccent.swift:10-18`; fix = key by
  `iconCacheKey` — one line, but interacts with PR-1's icon work; folded into
  REVIEW backlog.)

## 2. Performance

- **P1 [#46] Main-thread Trash scan on every rebuild** — `DockModel.rebuild`
  `opendir`s every candidate (including network/cloud volumes) on every app
  launch/quit/**activation**. A hung SMB share beach-balls every focus switch.
  Eliminated with the PR-1 icon fix.
- **P2 [#50] Per-keystroke waste in the Jetty Menu** —
  `JettyMenuModel.recompute` re-reads/JSON-decodes recents from UserDefaults and
  re-sorts the entire app index with ICU comparisons on *every keystroke*
  (`RecentAppsStore.swift:36-42`, `AppSearch.swift:50`); row icons are fetched
  synchronously from `NSWorkspace` into an unbounded dictionary; a fresh
  `NumberFormatter` per keystroke for currency. Fix: snapshot recents per show,
  reuse the index's own sort for the empty query, cache the formatter.
- **P3 [defer] Root-level hover state re-evaluates the whole DockView per mouse
  move** (`DockView.swift:18-19`): glass, decorations, CRT overlay, glows, all tiles
  re-evaluate per event; `tileCenters` is built twice per body; drag offsets are
  O(n²) per drag event. The PR-4 hover-region fix moves the tracker but the body
  granularity remains — needs a slots-subview hoist (REVIEW P1).
- **P4 [defer] 24/7 background churn**: the panel is always parked on-screen, so
  every `TimelineView` (now-playing spins up a fresh MediaRemote XPC controller
  every 5 s per display! `NowPlayingWidgetView.swift:12-17`), the 2 s system sampler
  (whose "off" gate is effectively always true — `DockController.swift:269-272`),
  and the window-peek 1 s re-capture loop all run while the dock is hidden. Needs
  the shared visibility coordinator from REVIEW P2/H4/H10.
- **P5 [defer] `DockStore.saveNow` does read-decode-copy-encode-write on the main
  queue** per slider tick (`DockStore.swift:159-195`). Needs the serial-I/O-queue
  design from REVIEW H5.
- **P6 [defer] `panel.invalidateShadow()` on every model rebuild** (every app
  activation, every display) — `DockPanelController.swift:472`.
- **P7 [#52] Network throughput ~2× on VPN** — `SystemStats.networkBytes` sums
  `utun/awdl/llw/ipsec/ppp/anpi` along with real interfaces. Fix: prefix filter.
- **P8 [#52] Timer tolerances** — 1 s/2 s repeating timers get near-zero tolerance;
  set `tolerance` (free power win).
- **P9 [defer] Window Peek re-enumerates + sequentially re-captures thumbnails every
  1.0 s while open** (`WindowPeek.swift:29-66`).
- **P10 [defer] Items settings decodes full-size images in row bodies, uncached**
  (`ItemsView.swift:131-162`); TrashMonitor re-enumerates all volumes per parent
  event (`TrashMonitor.swift:88-98`); weather/permissions timers run while hidden.
